NOXO EXPO ANDROID UYGULAMA GECIS KILAVUZU

Amac
Bu dosya, mevcut web tabanli NOXO projesinden Android uygulamaya gecis icin adim adim yol haritasi verir.
Hedefimiz ilk asamada:
- yeni bir Expo projesi acmak
- Android emulator veya telefonda calistirmak
- giris ekrani kurmak
- mevcut web backendine baglanmak
- ekranlari parca parca mobil tarafa tasimak

Onemli not
Bu ilk surecte web projesini kapatmiyoruz.
Web uygulamasi calismaya devam edecek.
Mobil uygulama ayri bir proje olacak.


1. GENEL YAKLASIM

Senin mevcut yapin:
- Web: Next.js
- Veri/Auth: Supabase
- UI: React tabanli dashboard

Mobilde yapacagimiz:
- Ayrica bir Expo React Native uygulamasi acacagiz
- Web tarafindaki veri ve auth mantigini yeniden kullanacagiz
- UI'yi mobil uyumlu sekilde bastan kuracagiz

Yani:
- backend buyuk oranda korunur
- is kurallari korunur
- ekranlar mobil icin yeniden yazilir


2. NEYI KURACAGIZ

Kurulacak temel araclar:
- Node.js LTS
- Android Studio
- Expo
- Expo Go veya Android emulator

Kontrol listesi:
- Bilgisayarda Node.js kurulu olmali
- npm calisiyor olmali
- Android Studio kurulu olmali
- Android SDK kurulu olmali
- Bir Android emulator olusturulmus olmali


3. ILK ON HAZIRLIK

Adim 1:
Terminal ac ve su komutlari kontrol et:

node -v
npm -v

Beklenen:
- Node surumu gorunmeli
- npm surumu gorunmeli

Node yoksa:
- Resmi Node.js LTS surumunu kur

Adim 2:
Android Studio kurulu mu kontrol et.

Kurulu degilse:
- Android Studio indir
- Kurulum sirasinda Android SDK, SDK Platform, Emulator bileşenlerini dahil et

Adim 3:
Android Studio icinde bir emulator olustur.

Onerilen:
- Pixel 7 veya benzeri
- Android 14 veya guncel stabil surum


4. PROJE STRATEJISI

Bu geciste iki repo modeli var:

Model A - En kolay baslangic:
- Mevcut NOXO web projesinin yanina ayri bir mobile klasoru acmak

Model B - Daha duzgun uzun vadeli:
- monorepo yapisi
- apps/web
- apps/mobile
- packages/shared

Senin icin ilk adimda onerim:
- once ayri bir Expo projesi ac
- sistem oturduktan sonra monorepo duzenine gec

Neden:
- ilk kez mobil yapiyorsun
- karmaasayi azaltir
- once urunu cikarirsin


5. ILK HEDEF: CALISAN BOS MOBIL UYGULAMA

Adim 1:
NOXO klasorunun bir ustunde veya yaninda yeni proje ac:

npx create-expo-app@latest noxo-mobile

Sorular gelirse:
- TypeScript sec

Adim 2:
Proje klasorune gir:

cd noxo-mobile

Adim 3:
Projeyi baslat:

npx expo start

Beklenen:
- tarayicida Expo arayuzu acilir
- terminalde QR ve secenekler cikar

Adim 4:
Android emulator acikken:
- terminalde a tusuna bas

Beklenen:
- uygulama emulatora yuklenir

Sorun olursa:
- Android Studio emulatoru acik mi kontrol et
- adb ve SDK kurulumunu kontrol et


6. ILK OGRENMEN GEREKEN TEMEL FARKLAR

Web tarafinda:

Ekranlari mobil mantikla yeniden kuracagiz.


7. KURMAN GEREKEN TEMEL PAKETLER

Expo projesi olustuktan sonra ilk kurulacaklar:

npx expo install @react-navigation/native
npx expo install react-native-screens react-native-safe-area-context react-native-gesture-handler react-native-reanimated
npx expo install @react-navigation/native-stack
npm install @supabase/supabase-js
npx expo install expo-secure-store

Bunlar ne ise yarar:
- navigation: ekran gecisleri
- supabase-js: auth ve veri
- secure-store: token saklama

9. ILK IS: SUPABASE BAGLANTISI

Adim 1:
Web projesindeki Supabase bilgilerini bul:
- NEXT_PUBLIC_SUPABASE_URL
- NEXT_PUBLIC_SUPABASE_ANON_KEY

Adim 2:
Expo projesinde env yapisi kur.
Baslangicta en basit yol:
- app config veya .env ile degiskenleri kullan

Adim 3:
Supabase client dosyasi olustur:
- src/lib/supabase.ts

Mantik:
- createBrowserClient kullanmayacaksin
- normal @supabase/supabase-js client kullanacaksin
- session bilgisini secure storage ile tutacaksin

Onemli:
Web'deki SSR cookie mantigini mobilde aynen kullanma.
Mobil ayri session mantigi ister.
